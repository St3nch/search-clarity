## Purpose

This checklist is used before delivering a SearchClarity customer-facing report.

It is channel-neutral and should be applied before Fiverr, Upwork, direct website, or other customer delivery.

---

## Core Rule

```text
No report is delivered until QC passes.
```

QC is not optional. It protects the customer, the brand, and the service quality standard.

No customer-facing delivery should be sent without human approval.

---

## QC Result Values

Use:

```text
passed
passed_with_notes
failed_requires_revision
blocked
```

- [ ] Internal work packet exists

- [ ] Biggest concern captured if provided

## 8. Visuals And Graphs Check

If the report includes visuals, confirm:

- [ ] Visuals clarify the report instead of decorating it
- [ ] Charts/graphs do not imply fake precision
- [ ] Priority matrices are explainable
- [ ] Visual labels use `observed`, `estimated`, or `SearchClarity estimate` language where appropriate
- [ ] Scorecards use plain-language ratings instead of fake numeric certainty unless a scoring method is documented
- [ ] Competitor tables are fair and within package scope
- [ ] Scorecards do not imply ranking, traffic, sales, or revenue certainty
- [ ] Labels are readable
- [ ] Visuals render correctly in the PDF/export
- [ ] Visuals do not expose private customer data

If no visuals are included, mark not applicable.

```text

```

## 11. Fiverr Gallery / Sample Asset Check

If the report or pages are used as Fiverr gallery/sample assets, confirm:

- [ ] Sample data is clearly labeled as sample, fictional, illustrative, or anonymized as appropriate
- [ ] No real customer/shop/listing identity appears without consent
- [ ] No external URLs appear in gallery images or sample PDF preview pages
- [ ] No QR codes appear in gallery assets
- [ ] No contact information appears in gallery assets
- [ ] No fake Fiverr badges, fake ratings, or fake trust seals appear
- [ ] No stock imagery or unowned assets are used
- [ ] Gallery image text is 10 words or fewer per image
- [ ] First 3 sample PDF pages work as a standalone proof asset
- [ ] Page 1 clearly communicates sample status
- [ ] Page 2 demonstrates executive summary quality
- [ ] Page 3 demonstrates priority/action-plan quality, preferably with effort/impact matrix

If not used as Fiverr gallery/sample material, mark not applicable.

```text

```

## 12. Report-To-Website Link Check

If the report links to SearchClarity website resources, glossary pages, methodology pages, or sample pages, confirm:

- [ ] Links work
- [ ] Links are useful to the buyer
- [ ] Links do not clutter the report
- [ ] Links do not route to unfinished/private pages
- [ ] Linked pages do not contain conflicting claims or pricing

Launch rule:

```text
Do not include glossary/resource links in launch reports or sample PDFs until SearchClarity.co pages are live, stable, and QC-approved.
```

If no report-to-website links are included, mark not applicable.

## 13. Video Walkthrough Check - If Included

If a short video report walkthrough is included, confirm:

- [ ] Video walkthrough is active for the purchased package/channel
- [ ] Recording outline/talking points reviewed
- [ ] Video matches the final report
- [ ] Video does not promise rankings, sales, traffic, revenue, or platform placement
- [ ] Video does not become coaching or ongoing support
- [ ] Video does not expose private customer data
- [ ] Captions are available or equivalent accessible text option exists
- [ ] Internal transcript generated
- [ ] Transcript reviewed for risky claims, contradictions, unclear statements, and privacy issues
- [ ] Video link/file works
- [ ] Video delivery reference recorded

If video is not included, mark not applicable.

## 16. Revision / Correction Check

Before delivery, confirm the revision/correction rules are clear.

- [ ] Customer revision count matches offer/package rules
- [ ] SearchClarity correction policy is understood
- [ ] SearchClarity mistakes do not count against buyer revision count
- [ ] Out-of-scope requests have a polite response path
- [ ] If a correction is already known, it is fixed before delivery


## 17. Raw Signal Capture Check

After report completion, ask:

- [ ] Did this order reveal a generalizable market/service pattern?
- [ ] If yes, was a raw signal logged?
- [ ] If no, was signal capture intentionally skipped?
- [ ] Does the signal avoid customer-identifying details?
- [ ] Does the signal avoid customer URLs, order IDs, and private report text?
- [ ] Is the signal captured without scoring the opportunity?
- [ ] Is customer-specific report content kept separate from generalized signal notes?

## Failed QC Handling

If QC fails:

1. Do not deliver.
2. Record required fixes.
3. Fix report.
4. Rerun QC.
5. Only deliver after pass.

---

## Non-Goals

This checklist does not:

- replace service-specific expertise
- replace legal review
- guarantee platform compliance
- replace fulfillment SOP
- authorize automated delivery
- authorize autonomous customer delivery
