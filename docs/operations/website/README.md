# Website Operations

## Purpose

This folder contains SearchClarity website planning and operations documents.

Use this folder for `searchclarity.co` planning, site architecture, SEO/GEO requirements, schema strategy, analytics setup, content planning, website repo workflow, and future controlled agent-assistance requirements.

This is SearchClarity-owned material.

It is not Neon Ronin doctrine.

## Current Domain

```text
searchclarity.co
```

## What Belongs Here

Use this folder for:

- website architecture plans
- hosting and stack decisions
- sitemap and page planning
- SEO/GEO/LLM visibility requirements
- schema strategy
- blog/content strategy
- analytics setup notes
- Search Console/Bing Webmaster setup notes
- website repo workflow
- internal linking plans
- page refresh/update process
- website-specific proof asset planning
- future agent-assisted site optimization requirements

## What Does Not Belong Here

Do not put these here:

- website source code
- exported reports or PDFs
- customer files
- Neon Ronin agent runtime implementation
- database architecture
- generic multi-business doctrine
- Fiverr-specific gig copy
- fulfillment SOPs unrelated to website operations

Website implementation code should live in a future separate repo, likely:
